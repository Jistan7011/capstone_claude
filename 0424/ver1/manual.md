# 라인 트레이서 사용 설명서

## 시스템 구성

```
[웹 브라우저] ──HTTP──▶ [Jetson Nano : Flask 서버 + 카메라]
                                  │
                              UART0 (115200 baud)
                                  │
                         [ATmega128 : 모터 제어]
                           ├─ UART1 ── 블루투스 모듈
                           ├─ I2C  ── PN532 RFID 리더
                           └─ PWM  ── 좌/우 모터
```

---

## 1. 시작 방법

### ATmega128 펌웨어 빌드 및 업로드

```bash
cd atmega128
make                          # 컴파일 → line_tracer.hex 생성
avrdude -p atmega128 -c usbtiny -U flash:w:line_tracer.hex
```

### Jetson Nano 서버 실행

```bash
# 의존성 설치 (최초 1회)
pip install -r jetson_nano/requirements.txt

# 서버 시작
python jetson_nano/app.py
```

서버가 시작되면 터미널에 포트 번호가 출력된다.

### 대시보드 접속

- **Jetson 로컬**: `http://127.0.0.1:8080`
- **같은 네트워크**: `http://<Jetson_IP주소>:8080`

## 2. 환경 변수 설정 (선택)

`.env.example`을 참고해 환경 변수로 동작을 바꿀 수 있다.

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `CAM_INDEX` | `0` | 카메라 디바이스 번호 |
| `HTTP_PORT` | `8080` | 웹 서버 포트 |
| `SERIAL_PORT` | `/dev/ttyUSB0` | ATmega 연결 시리얼 포트 |
| `SERIAL_ENABLED` | `1` | `0`으로 설정하면 시리얼 없이 실행 (PC 테스트용) |
| `AUTO_ON_START` | `1` | 서버 시작 시 자동으로 AUTO 모드 전환 |

```bash
# 예시: 시리얼 없이 PC에서 테스트
SERIAL_ENABLED=0 python jetson_nano/app.py
```

## 3. 웹 대시보드 사용법

### 화면 구성

```
┌──────────────────────────────┬─────────────────────┐
│  Live View (실시간 영상)      │  ATmega Live Status  │
│  [AUTO]  [MANUAL]            │  (상태 카드)          │
│  [LEFT] [FORWARD] [RIGHT]    ├─────────────────────┤
│  [STOP]                      │  Location (RFID 구역) │
│  [저속] [중속] [고속] [최대]  │                      │
├──────────────────────────────┴─────────────────────┤
│  Recent Commands (최근 명령 이력)                    │
├─────────────────────────────────────────────────────┤
│  Serial Log (시리얼 로그)                            │
└─────────────────────────────────────────────────────┘
```

### 모드 버튼

| 버튼 | 전송 명령 | 동작 |
|------|-----------|------|
| **AUTO** | `A` | 자동 모드: Jetson이 카메라로 라인을 감지해 자동 주행 |
| **MANUAL** | `M` | 수동 모드: 웹/블루투스 버튼으로 직접 제어 |

현재 활성 모드의 버튼은 **파란색**으로 표시된다.

### 방향 버튼 (수동 모드에서 사용)

| 버튼 | 전송 명령 | ATmega 동작 |
|------|-----------|------------|
| **FORWARD** | `F` | 양쪽 모터 전진 |
| **LEFT** | `L` | 좌회전 (왼쪽 절반 속도) |
| **RIGHT** | `R` | 우회전 (오른쪽 절반 속도) |
| **STOP** | `S` | 모터 정지 |

현재 방향 버튼도 ATmega 텔레메트리 기준으로 파란색으로 표시된다.

> AUTO 모드에서 방향 버튼을 누르면 ATmega에 전달되지만, Jetson의 라인 감지가 계속 명령을 덮어쓴다.

### 속도 버튼 (수동 모드 전용)

| 버튼 | 속도값 | 설명 |
|------|--------|------|
| **저속** | 75 | 천천히 이동 |
| **중속** | 125 | 기본 속도 |
| **고속** | 175 | 빠르게 이동 |
| **최대** | 255 | 최대 속도 |

- AUTO 모드에서는 속도 버튼이 **흐리게 표시**되며 클릭되지 않는다.
- 현재 속도는 버튼 그룹 레이블 옆 숫자로 표시된다.

## 4. 블루투스 제어 (시리얼 터미널)

ATmega128의 UART1(블루투스)에 연결된 터미널에서 직접 제어할 수 있다.

### 모드 전환

| 키 | 동작 |
|----|------|
| `A` 또는 `a` | AUTO 모드 전환 |
| `M` 또는 `m` | MANUAL 모드 전환 |
| `스페이스` | 긴급 정지 (MANUAL 강제 전환 + 모터 즉시 정지) |

### 방향 제어 (MANUAL 모드)

| 키 | 동작 |
|----|------|
| `w` / `W` | 전진 |
| `s` / `S` | 후진 |
| `l` / `L` | 좌회전 |
| `r` / `R` | 우회전 |
| `x` / `X` | 정지 |

### 속도 제어 (MANUAL 모드)

| 키 | 속도값 | 설명 |
|----|--------|------|
| `1` | 25 | |
| `2` | 50 | |
| `3` | 75 | |
| `4` | 100 | |
| `5` | 125 | |
| `6` | 150 | |
| `7` | 175 | |
| `8` | 200 | |
| `9` | 225 | |
| `0` | 255 | 최대 |
| `+` | 현재+10 | 10 단계 증가 |
| `-` | 현재-10 | 10 단계 감소 |

## 5. 상태 카드 설명

### ATmega Live Status

| 항목 | 설명 |
|------|------|
| `camera_opened` | 카메라 정상 연결 여부 (OK/FAIL) |
| `serial_connected` | ATmega 시리얼 연결 여부 (OK/FAIL) |
| `current_direction` | 현재 모터 방향 (FORWARD/LEFT/RIGHT/STOP) |
| `mode` | 현재 모드 (AUTO/MANUAL) |
| `line_tracer` | 라인 감지 명령 전송 상태 (ACTIVE/PAUSED) |
| `rpm_left` | 왼쪽 모터 RPM |
| `rpm_right` | 오른쪽 모터 RPM |
| `manual_speed` | 수동 모드 현재 속도값 (0–255) |
| `location` | RFID로 감지된 현재 구역 |
| `telemetry_age` | 마지막 텔레메트리 수신 경과 시간 |
| `vision_state` | 라인 감지 상태 (ACTIVE/IDLE) |
| `vision_err` | 라인 중심과 화면 중앙의 픽셀 오차 |

### Location 카드

RFID 태그를 차량이 통과할 때 구역명이 크게 표시된다.

| 구역명 | 설명 |
|--------|------|
| A구역 | RFID 태그 A |
| B구역 | RFID 태그 B |
| C구역 | RFID 태그 C |
| D구역 | RFID 태그 D |
| `-` | 감지된 구역 없음 |

## 6. 모드별 동작 정리

### AUTO 모드

1. Jetson이 카메라로 바닥의 검은 라인을 감지한다.
2. 라인 중심과 화면 중앙의 오차(±40px 데드밴드)를 계산해 F/L/R/S 명령을 ATmega로 80ms 간격으로 전송한다.
3. ATmega가 명령을 300ms 이상 수신하지 못하면 자동으로 정지한다.
4. 속도: 직진 120, 회전 시 안쪽 60 (하드코딩)

| 상황 | Jetson 명령 |
|------|-------------|
| 라인이 중앙 ±40px 이내 | `F` (직진) |
| 라인이 왼쪽으로 치우침 | `L` (좌회전) |
| 라인이 오른쪽으로 치우침 | `R` (우회전) |
| 3프레임 연속 라인 미감지 | `S` (정지) |

### MANUAL 모드

- Jetson은 영상 스트리밍을 계속하지만 라인 감지 명령 전송을 중지한다.
- 웹 대시보드 방향 버튼 또는 블루투스로 직접 제어한다.
- 속도는 웹 속도 버튼(`/speed/<level>`) 또는 블루투스 숫자키로 조절한다.

## 7. API 엔드포인트 (직접 호출)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/` | 대시보드 HTML |
| `GET` | `/video_feed` | MJPEG 영상 스트림 |
| `GET` | `/health` | 전체 상태 JSON |
| `POST` | `/send/<cmd>` | 명령 전송 (cmd: F/L/R/S/A/M) |
| `POST` | `/speed/<level>` | 속도 설정 (level: 0–9) |

```bash
# 예시: curl로 명령 전송
curl -X POST http://127.0.0.1:8080/send/A      # AUTO 모드
curl -X POST http://127.0.0.1:8080/send/M      # MANUAL 모드
curl -X POST http://127.0.0.1:8080/speed/5     # 속도 125 설정
curl http://127.0.0.1:8080/health              # 현재 상태 조회
```

## 8. 문제 해결

| 증상 | 확인 사항 |
|------|-----------|
| `serial_connected: FAIL` | USB 시리얼 케이블 연결 확인, `SERIAL_PORT` 환경 변수 확인 (`ls /dev/ttyUSB*`) |
| `camera_opened: FAIL` | 카메라 연결 확인, `CAM_INDEX` 값 변경 시도 (0, 1, 2) |
| 영상이 안 보임 | 브라우저에서 `/video_feed` 직접 접속해 MJPEG 스트림 확인 |
| 모터가 반응 없음 | ATmega 전원 및 UART0 연결 확인, `Serial Log`에서 오류 메시지 확인 |
| 라인을 감지 못함 | 조명 조건 확인, `vision_err` 값이 계속 null이면 카메라 각도 조정 |
| AUTO 모드인데 계속 정지 | 라인 미감지 3프레임 조건 — 카메라 ROI(하단 35%) 안에 라인이 있는지 확인 |
| RFID 구역이 표시 안 됨 | PN532 I2C 연결 확인, 태그 UID가 펌웨어에 등록된 4개 중 하나인지 확인 |
